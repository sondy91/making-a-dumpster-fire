# Making a Dumpster Fire 🔥🛠️

This repository contains the blueprints, build-scripts, and documentation used to create the legendary [Dumpster Fire Driven Development (DFDD)](https://github.com/sondy91/dumpster-fire-driven-development) project.

## What is this?
While the main DFDD repository serves as the "Enterprise Monolith" (the actual 35-million-line liability), this repository holds the **meta-infrastructure** required to generate that chaos. 

## Included Assets:
*   **`scripts/`**: The "Bloat Engine", "Commit Spammer", and "Config Graveyard" generators that programmatically manufactured the 35M LOC and the 1,000 atomic commits.
*   **`blog-posts/`**: The original draft of the viral blog post: "How I Wrote 35 Million Lines of Code Over the Weekend".
*   **`PLAN.md`**: The chronological record of how the project was brainstormed, from UI Hydra modals to the "Do Not Press" button.

## Why separate them?
In true Enterprise fashion, we have separated the "Business Logic" (the dumpster fire) from the "DevOps & Infrastructure" (the match that lit it). This allows for maximum architectural synergy and prevents the build-scripts from being lost in the 35 million lines of validation code.
